FLUJO COMPLETO END-TO-END
═════════════════════════════════════
1. User clicks "Solicitar acceso" button
   → DataRequestButton triggers creation
   → POST /data-requests (request_type: 'access')
   → status='pending', "En espera" badge displays

2. Polling begins automatically every 5 seconds
   → GET /data-requests
   → DataRequestRow updates status badge
   → pending → processing → completed

3. On completion, auto-download triggers
   → GET /data-requests/{id}/download
   → Blob received (application/json)
   → Filename: data_export_2026-05-29_143050.json
   → Silent download, no user action required
   → URL cleanup via setTimeout revoke

4. Analytics events batched and posted
   → request_created, request_completed, export_downloaded
   → 10-event threshold or 30-second interval
   → POST /api/v1/analytics/events with Bearer token
   → On 401: refresh token, re-enqueue to head, retry

5. User can cancel at any point
   → Click "Cancelar" button
   → ConfirmationModal opens
   → Variant: 'warning' for cancel, 'danger' for deletion
   → Click "Continuar"
   → POST /data-requests/{id}/cancel
   → request_cancelled analytics event tracked
   → Modal closes, success message displays
