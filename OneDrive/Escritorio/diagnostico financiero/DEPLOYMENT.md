# Deployment Checklist — Render.com

## Pre-Deployment (Local)

**Run validation:**
```bash
bash validate-build.sh
```

**Manual tests:**
```bash
# Terminal 1: Install and build
npm install --legacy-peer-deps
npm run build

# Terminal 2: Start server
python app_standalone.py
```

Then:
- Open http://localhost:8000
- Load a few questions (should fetch from `/api/v1/schema`)
- Answer 2-3 questions
- Verify back button works
- Submit and check PDF generation

## Render Deployment

1. Go to https://dashboard.render.com
2. **New → Web Service**
3. **Select GitHub repo** → `diagnostico-financiero`
4. **Auto-detect from render.yaml:**
   - Name: `diagnostico-financiero-v6`
   - Runtime: Python 3.11
   - Build Command: `./build.sh`
   - Start Command: `python app_standalone.py`

5. **Wait for build** (3-5 minutes):
   - Install npm dependencies
   - Build React app → dist/
   - Install pip dependencies
   - Start FastAPI

## Post-Deployment Validation

Once deployment finishes:

1. **Check live URL** (from Render dashboard)
   - Should load React app
   - Network tab should show `/api/v1/schema` working

2. **Test API endpoints:**
   ```bash
   curl https://YOUR_RENDER_URL/health
   curl https://YOUR_RENDER_URL/api/v1/schema | head -100
   ```

3. **Test full flow:**
   - Load questionnaire
   - Answer 10+ questions
   - Submit and verify PDF download

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `vite: not found` | npm install didn't complete. Re-run: `npm install --legacy-peer-deps` |
| `dist/ not found` | npm run build failed. Check build logs for missing dependencies |
| Port already in use | Change PORT env var: `PORT=3000 python app_standalone.py` |
| React app won't load | Check FastAPI logs for 404 on `/api/v1/schema` |
| PDF generation fails | Verify ReportLab in requirements.txt and check reports/ directory |

## Key Architecture

```
┌─────────────────────────────────────────┐
│  Browser (React SPA)                    │
│  ┌─────────────────────────────────┐   │
│  │ QuestionnaireFlow (500 questions)   │   
│  │ - useCallback for handlers          │
│  │ - React.memo for child components   │
│  │ - O(1) question lookup (Map)        │
│  │ - 100ms debounce for concurrency    │
│  └─────────────────────────────────┘   │
└──────────────────┬──────────────────────┘
                   │ HTTP
┌──────────────────┴──────────────────────┐
│  FastAPI Server (Single process)        │
│  ┌─────────────────────────────────┐   │
│  │ GET /api/v1/schema              │   │
│  │  → Load 500 questions            │   │
│  ├─────────────────────────────────┤   │
│  │ POST /api/v1/diagnose           │   │
│  │  → Score answers                │   │
│  │  → Generate PDF                 │   │
│  │  → Return report path           │   │
│  ├─────────────────────────────────┤   │
│  │ StaticFiles(dist/)              │   │
│  │  → Serve React app              │   │
│  │  → SPA fallback to index.html    │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

## Memory Leak Fixes Applied

1. **Event listener accumulation** → All handlers wrapped in `useCallback`
2. **O(n) question lookups** → Memoized Map structure for O(1) access
3. **Concurrent clicks** → `isAnswering` guard + 100ms debounce
4. **Cascade re-renders** → `React.memo` on all child components
5. **Missing back button** → `questionHistory` array tracking
